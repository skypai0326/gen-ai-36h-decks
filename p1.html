Concept · 工具沒有最強，只有最對

「哪個最強」
是假議題

就像問「哪把刀最厲害」——生魚片刀削魚順手、菜刀拍蒜頭快，
沒有「最強」，只有「這件事最適合」。

每個 AI 品牌把研發資源壓在不同強項。
你需要的不是「選一家」，是每種任務知道找誰的直覺。

Resolution Compare · 18 字 vs 72 字

✗ 新手 · 18 字

→ 套話開頭、模糊原因、敬請見諒。花 15 分鐘改才能寄。

✓ 公式 · 72 字
你是專案經理。寫中文 email 給王小姐，告知 A 案因供應商缺料延後 5 天。語氣正式有同理心，含 2 個補救方案，200 字內。

→ 角色感、具體事實、可選方案、字數合理。檢查一眼就能寄。

Role & Task · 同件事的「差 / 普通 / 好」

角色越具體，產出越專業

✗ 差
「你是行銷」 · 一句話「處理一下這份報告」
通用 AI 口吻
△ 普通
「你是行銷顧問」 · 「幫我整理這份報告」
有點專業感
✓ 好
「你是 5 年經驗 B2B SaaS 行銷顧問，擅長科技業採購流程」 · 「把 30 頁 Q3 銷售報告濃縮成 5 點給董事會」
精準對焦
動詞清單：寫 / 摘要 / 改寫 / 分類 / 萃取 / 翻譯 / 評估 / 比較。
避免「處理一下」「看看這個」「幫我弄」。

Pipeline · 按 → 逐版加料 · 看 prompt 一層層長出來

同一 Prompt，10 字長到 130 字的 5 版

v.1
10 字

v.2
+ Role
+「你是 8 年資歷專案經理」 → 口吻變專業，內容仍空

+ Task
+ 王小姐 / A 案 / 缺料 / 5 天 → 事實到位，語氣偏書面

+ Format
+ 主旨 / 內文 / 署名範本 → 複製貼到 Gmail 就能寄

1-3 · Two Layers
18 / 26 講義 · CH1-3
Two Layers · 永久身份 vs 單次任務

System 是長期身份，
User 是當下任務

System Prompt · 設一次
員工的職務說明書

放不會變的東西：角色、語氣、產業背景、固定格式、禁忌、知識邊界。寫進去後該對話視窗 / Project / Gem 每次提問都自動套用。

User Prompt · 每次寫
主管的每日工作指令

放每次都變的東西：本次任務、當次限制（字數、受眾）、這次想要的格式。

判斷口訣：「下一次用還會一樣嗎？」 一樣 → System；每次變 → User。

Act III · Setup ★ Open Lecture Notes
1-3 · Email Assistant Template
20 / 26 講義 · CH1-3
Live Template · 貼進 Claude Projects 直接用

Email 助理 · System Prompt 範本

# 角色　你是 5 年資歷的行政專員，協助寫中文商務 email。
# 固定原則　語氣正式 + 同理心｜開頭直接切入｜結尾「謝謝，[姓名]」｜200 字內｜指令模糊先問清。
# 禁用詞　首先 · 其次 · 綜上所述 · 敬請查照 · 不便之處敬請見諒 · 萬分抱歉 · 煩請撥冗 · 惠請 · 謹此。
# 我的身份　王小明｜弄一下工作室 業務經理｜對象：客戶 / 供應商 / 同事。
# 流程　① 讀懂貼上的客戶信　② 按上述原則寫回信　③ 附「若要調整請告訴我」。

過去：每次貼 300 字 prompt + 100 字客戶信 = 400 字。
現在：「幫我回這封」+ 100 字客戶信 = 110 字。一週省 50 分鐘。
